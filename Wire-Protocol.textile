This page documents the details of the low-level wire protocol that Cucumber uses to list and invoke steps in a remote process.

*{color:red}This protocol is in flux. Please see "wire protocol":https://rspec.lighthouseapp.com/projects/16211/tickets/428-wire-protocol for discussions around this emerging protocol.*

h2. List Step Definitions

<pre>
|-- list_step_definitions --->|
|<-------------------- ok --|
</pre>

h3. Request 

<pre>{
  "version" : "1.0",
  "list_step_definitions" : ""
}</pre>
h3. Response

<pre>{
  "version" : "1.0",
  "step_definitions": [
    { "id": "id1", "regexp": "regex1" },
    { "id": "id2", "regexp": "regex2" }
  ]
}</pre>
h2. Invoke

<pre>
|-- invoke --------->|
|<------------- ok --|
         OR
|<----- diff_table --|
|-- diff_response -->|
|<------------- ok --|
         OR
|<---- step_failed --|

</pre>


h3. Request 

<pre>{
  "version" : "1.0",
  "invoke" : "id1",
  "params" : [ "foo", "bar" ]
}</pre>
h3. Response

If the step is invoked successfully then an 'ok' response is returned:

<pre>{
  "version" : "1.0",
  "ok" : ""
}</pre>

OR If the step throws an exception then a 'step_failed' response is returned:

<pre>{
  "version" : "1.0",
  "step_failed" : {
    "exception" : "some exception type",
    "message" : "foo",
    "backtrace" : "bar"
  }
}</pre>

h2. Table Diff

This message is sent by the wire server when it requires Cucumber to to a table diff when invoking a step.

<pre>
|<------------ diff_table --|
|-- diff_response --------->|
</pre>

h3. Request 

<pre>{
  "version" : "1.0",
  "diff_table" : "TBD",
}</pre>
h3. Response

<pre>{
  "version" : "1.0",
  "diff_response" : "TBD"
}</pre>


h2. General Failure Response

If the remote process cannot handle a request then it shall return a 'failed' response:

<pre>{
  "version" : "1.0",
  "failed" : {
    "exception" : "some exception type",
    "message" : "foo",
    "backtrace" : "bar"
  }
}</pre>
